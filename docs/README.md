## Documentation Principles

This documentation follows best practices from *The Pragmatic Programmer*:

1. **Single Source of Truth** - Each piece of information lives in one place
2. **Progressive Disclosure** - Start simple, dive deeper as needed
3. **Task-Oriented** - Organized by what you want to accomplish
4. **Maintainable** - Clear structure makes updates easy
5. **Discoverable** - Multiple paths to find what you need

## Contributing to Documentation

When adding or updating documentation:

1. Place files in the appropriate category directory
2. Use clear, descriptive filenames
3. Update this README if adding new documents
4. Follow existing formatting conventions
5. Keep related content together
